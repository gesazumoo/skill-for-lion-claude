# 프로젝트 용어 정의

이 프로젝트에서는 아래 용어를 기준으로 작업해줘.

같은 의미를 다른 이름으로 새로 만들지 말고,
기존 용어를 최대한 유지해서 작업해줘.

---

# 클래스

사용자가 탐색하고 신청할 수 있는 콘텐츠 단위.

예:
- 운동 클래스
- 뜨개질 클래스
- 수영 클래스
- 러닝 클래스
- 스터디 클래스

---

# 클래스 정보

각 클래스가 가지고 있는 정보.

포함 정보:
- 클래스 제목
- 클래스 카테고리
- 가격
- 지역
- 날짜
- 참가 가능 인원
- 참가 신청 인원
- 썸네일 이미지
- 마감 날짜
- 상세 설명

---

# 클래스 카드

클래스 정보를 카드 형태로 보여주는 UI.

기본 표시 정보:
- 클래스 제목
- 클래스 카테고리
- 가격
- 지역
- 날짜
- 참가 인원
- 썸네일 이미지
- 마감 임박 여부
- 신청하기 버튼

마감 임박 여부는:
마감 날짜 기준 3일 이내 남았을 경우 표시.

---

# 가로형 클래스 카드

가로 스크롤 클래스 리스트에서 사용하는 카드 형태.

빠르게 탐색할 수 있는 간단한 구조로 구성.

표시 정보:
- 썸네일 이미지
- 클래스 제목
- 가격
- 마감 임박 여부
- 참가 인원

---

# 세로형 클래스 카드

세로 리스트에서 사용하는 카드 형태.

정보를 조금 더 자세하게 보여주는 구조로 구성.

표시 정보:
- 썸네일 이미지
- 클래스 제목
- 클래스 카테고리
- 날짜
- 지역
- 참가 인원
- 가격
- 신청하기 버튼

---

# 클래스 리스트

여러 개의 클래스 카드를 리스트 형태로 보여주는 영역.

종류:
- 가로 스크롤 클래스 리스트
- 세로 스크롤 클래스 리스트

---

# 클래스 상세보기

선택한 클래스의 상세 정보를 보여주는 팝업 UI.

표시 정보:
- 클래스 제목
- 클래스 카테고리
- 가격
- 지역
- 날짜
- 참가 가능 인원
- 참가 신청 인원
- 썸네일 이미지
- 마감 날짜
- 상세 설명

모바일에서 보기 좋은 형태로 구성.

---

# 하단 네비게이션

앱 하단에 고정된 메뉴 UI.

메뉴:
- 홈
- 검색
- 등록
- 내정보

현재 활성화된 메뉴는 active 상태로 표시.

---

# 홈 화면

사용자가 서비스에 처음 들어왔을 때 보는 메인 화면.

추천 클래스, 마감 임박 클래스 등을 탐색할 수 있는 화면.

---

# 검색 화면

사용자가 클래스 검색과 카테고리 탐색을 할 수 있는 화면.

클래스 리스트와 클래스 상세보기를 포함.

---

# 등록 화면

사용자가 새로운 클래스를 등록할 수 있는 화면.

입력 폼 중심 구조로 구성.

---

# 내정보 화면

사용자의 정보와 활동 내역을 확인할 수 있는 화면.

신청 내역, 프로필 정보 등을 포함.

---

# 더미 데이터

API 또는 DB 연결 전 화면 구현을 위해 사용하는 임시 데이터.

화면이 비어보이지 않도록 실제 서비스처럼 구성.

---

# 컴포넌트 재사용

이미 만든 UI와 기능이 있다면 새로 만들지 말고 최대한 재사용해서 작업.

특히 아래 요소는 공통으로 재사용:
- 클래스 카드
- 클래스 리스트
- 하단 네비게이션
- 버튼 스타일
- 입력창 스타일
