## 작업 전 참고

작업 전 아래 문서를 먼저 참고해줘.

- ./project-rules/glossary.md
- ./project-rules/workflow.md

프로젝트 용어와 작업 원칙은 해당 문서를 기준으로 작업해줘.
현재 프로젝트는 Supabase 연결이 완료된 상태이며,
등록된 데이터가 실제 DB에 저장되도록 구현해줘.
---

## 화면 목적

사용자가 새로운 클래스를 직접 등록할 수 있는 등록 화면을 구현해줘.

---

## 기존 화면과 연결

하단 푸터에서 등록 누르면 이 화면으로 연결하면 돼


---

## 화면 구성


### 1. 상단 영역

포함 요소:
- 화면 제목
- 간단한 안내 문구

예:
"새로운 클래스를 등록해보세요"

---

### 2. 클래스 정보 입력 영역

사용자가 클래스 정보를 입력할 수 있는 입력 폼을 만들어줘.

입력 항목은 glossary.md의 클래스 정보 기준을 따라줘.

사용하기 쉬운 모바일 입력 UI로 구성해줘.

---

### 3. 썸네일 업로드 영역

사용자가 클래스 썸네일 이미지를 등록할 수 있는 UI를 만들어줘.

우선은 간단한 업로드 UI 형태로 구현해줘.

---

### 4. 등록 버튼 영역

포함 요소:
- 등록하기 버튼
- 취소 버튼

모바일에서 누르기 편한 형태로 구성해줘.

---

## 필요한 기능

아래 기능들이 실제로 동작하게 구현해줘.

- 카테고리 선택
- 날짜 입력
- 이미지 업로드 UI
- 등록 버튼 클릭
- Supabase 연결
- 등록 성공 처리
- 입력값 검증 처리
- 로딩 상태 처리

등록 성공 시:
- 성공 메시지 표시
- 홈 화면 / 검색 화면에서 데이터 확인 가능하도록 연결

---

## 구현 조건

- Supabase 사용
- 모바일 반응형 우선
- 기존 디자인 스타일 유지
- 기존 화면이 깨지지 않게 구현
- 기존 컴포넌트 최대한 재사용
- 불필요한 라이브러리 추가 금지

현재 단계:
- API 연결 여부: O
- DB 연결 여부: O
- 더미 데이터 사용 여부: 제거됨

---

## 추가 조건

- 실제 서비스처럼 자연스럽게 동작하게 구현해줘.
- 입력 흐름이 복잡하지 않도록 구성해줘.
- 모바일 앱 느낌으로 구현해줘.

---

## 작업 후 정리

작업이 끝나면 CLAUDE.md에 아래 내용을 정리해줘.

- 새로 만든 화면
- 추가된 입력 기능
- 연결된 DB 테이블
- 추가된 등록 기능
- 다음 작업자가 참고해야 할 내용

의미 있는 작업이므로 사용한 프롬프트를 `/prompts` 폴더에 저장해줘.
