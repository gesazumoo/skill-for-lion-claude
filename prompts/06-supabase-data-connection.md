## 작업 목적

현재 화면에서 사용중인 더미 데이터를 실제 Supabase 데이터로 연결해줘.

기존 UI와 기능은 유지하면서 실제 DB 데이터를 사용하는 구조로 변경하는 단계야.

## 기존 화면과 연결

현재 구현되어 있는 화면 구조와 UI는 유지해줘.


---

## 작업 내용

### 1. Supabase 테이블 연결

현재 화면에서 사용중인 데이터 구조를 기준으로
Supabase 테이블 구조를 확인하거나 생성해줘.

기존 더미 데이터 구조를 최대한 유지해줘.

---

### 2. 목록 조회 연결

현재 더미 데이터로 표시되던 항목들을
Supabase 데이터 조회 기반으로 변경해줘.

기존 UI와 컴포넌트는 유지해줘.

---

### 3. 상세보기 연결
현재 화면에서 구현되어 있는 기능들이

Supabase 데이터 기반에서도 동일하게 동작하도록 유지해줘.

예:

- 상세보기

- 검색

- 필터

- 상태 표시

- 정렬

- 탭 전환

기존 사용자 경험과 UI 흐름은 최대한 유지해줘.

---

## 필요한 기능

- 로딩 상태 처리
- 데이터 없을 때 예외 처리

---

## 구현 조건

- 기존 화면 구조 최대한 유지
- 기존 디자인 스타일 유지
- 기존 컴포넌트 최대한 재사용
- 모바일 반응형 유지
- 기존 화면이 깨지지 않게 구현
- 불필요한 라이브러리 추가 금지

현재 단계:
- API 연결 여부: Supabase 연결 O
- DB 연결 여부: Supabase 연결 O
- 더미 데이터 사용 여부: 제거 예정

---

## 추가 조건

- 우선은 조회(Read) 기능만 구현해줘.
- 등록(Create), 수정(Update), 삭제(Delete)는 아직 구현하지 않아도 돼.
- 기존 더미 데이터 구조와 최대한 비슷하게 DB 데이터를 연결해줘.
