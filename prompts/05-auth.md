현재 내정보 메뉴는 로그인 상태와 연결되도록 구현해줘.

---

## 작업 내용

### 1. Supabase Auth 연결

Supabase Auth의 이메일/비밀번호 방식으로 회원가입과 로그인을 구현해줘.

포함 기능:
- 회원가입
- 로그인
- 로그아웃
- 로그인 상태 유지
- 회원가입 후 이메일 인증 안내 메시지 처리

Supabase 프로젝트 설정에서 Email Auth가 활성화되어 있다는 전제로 구현해줘.

---

### 2. 로그인 / 회원가입 화면 구현

모바일 앱 느낌의 인증 화면을 구현해줘.

포함 요소:
- 이메일 입력
- 비밀번호 입력
- 로그인 버튼
- 회원가입 버튼
- 로그인 / 회원가입 화면 전환 UI
- 인증 안내 메시지
- 에러 메시지

너무 복잡하지 않고 실제 서비스처럼 자연스럽게 구성해줘.

---

### 3. 로그인 상태 처리

로그인 상태를 전역적으로 관리해줘.

포함 기능:
- 현재 로그인 상태 확인
- 로그인 여부에 따라 UI 변경
- 새로고침 후 로그인 유지
- 로그아웃 처리

---

### 4. 내정보 화면 연결

내정보 화면에서 로그인 상태에 따라 아래처럼 동작하게 구현해줘.

비로그인 상태:
- 로그인 안내 화면 표시
- 로그인 버튼 표시
- 회원가입 버튼 표시

로그인 상태:
- 사용자 이메일 표시
- 사용자 정보 영역 표시
- 로그아웃 버튼 표시
- 이후 내 활동 정보를 추가할 수 있는 구조로 구성

---

## 필요한 기능

아래 기능들이 실제로 동작하게 구현해줘.

- 이메일/비밀번호 회원가입
- 이메일/비밀번호 로그인
- 로그아웃
- 로그인 상태 유지
- 로그인 상태에 따른 내정보 화면 분기
- 회원가입 후 이메일 인증 안내 메시지 표시
- 인증 실패 처리
- 로딩 상태 처리
- 기본 에러 메시지 처리
